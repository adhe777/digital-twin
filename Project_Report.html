Project Abstract
Digital Twin for Personal Productivity

Project Abstract

The "Digital Twin for Personal Productivity" is a software system designed to model and optimize the user's daily academic and professional life. By creating a digital counterpart of the user, the system tracks routines, sleep patterns, study habits, and productivity metrics. It leverages data analytics to provide actionable insights, enabling users to balance their "Student" and "Professional" roles effectively. The system facilitates goal setting, routine management, and performance tracking, acting as a comprehensive tool for self-improvement and time management.

1. Database Structure

TABLE NAME: users
Description: Stores the personal details and authentication credentials of the system users.

Sl no Field Type Description
1 user_id ObjectId Primary Key. Unique identifier for the user.
2 name VARCHAR(100) Full name of the user.
3 email VARCHAR(150) Unique email address for login.
4 role ENUM User role ('Student' or 'Professional').

TABLE NAME: routines
Description: Stores daily activity logs including sleep, study, and screen time metrics.

Sl no Field Type Description
1 routine_id ObjectId Primary Key. Unique identifier for the routine entry.
2 user_id ObjectId Foreign Key referencing users table.
3 sleep_hours FLOAT Number of hours slept.
4 study_hours FLOAT Number of hours spent studying.
5 screen_time FLOAT Hours of screen usage.
6 mood INT Self-reported mood rating (1-5).

TABLE NAME: productivity_scores
Description: Stores the calculated productivity scores based on routine data.

Sl no Field Type Description
1 score_id ObjectId Primary Key.
2 user_id ObjectId Foreign Key referencing users table.
3 score INT Calculated productivity score.
4 date DATETIME Date of the score calculation.

TABLE NAME: goals
Description: Stores specific academic or professional goals set by the user.

Sl no Field Type Description
1 goal_id ObjectId Primary Key.
2 user_id ObjectId Foreign Key referencing users table.
3 category_id ObjectId Foreign Key referencing categories (Shared Core).
4 target_date DATE Deadline for the goal.
